# Product AI Crew

> PM · 기획자 · 디자이너 · UX라이터로 구성된 AI 멀티 에이전트 시스템

---

## 한 줄 소개

조직에 없는 역할을 AI 에이전트로 채워, 혼자서도 전체 프로덕트 파이프라인을 돌릴 수 있는 시스템입니다.

---

## 만든 이유

### 디자인 업무를 하면서 느낀 역할 공백

UX 디자이너로 일하면서 "이 기능을 왜 만들어야 하는가"를 정의하는 PM도, 화면 흐름을 체계적으로 기획하는 기획자도 없는 상황을 자주 마주쳤습니다. 결국 디자인 이전 단계부터 모두 혼자 정리해야 했습니다.

### UX 라이팅 봇을 만들다가 생각이 확장된 과정

처음에는 Claude Code로 UX 라이팅 전용 에이전트를 만들어 사용했습니다. 반복 작업이 눈에 띄게 줄어드는 걸 경험하면서 "UX 라이팅만이 아니라 그 위 단계들도 AI로 채울 수 있지 않을까?"라는 생각으로 이어졌습니다.

### 조직에 없는 역할을 AI로 채우게 된 계기

작은 팀일수록 역할 구분 없이 넓게 쳐야 합니다. 각 역할의 산출물 형식과 인풋/아웃풋 흐름을 명확하게 정의하면 AI 에이전트도 협업 동료처럼 동작할 수 있다는 것을 확인했고, 그게 이 시스템의 출발점입니다.

## 구성된 에이전트

![에이전트 구성](assets/agents.png)

| 에이전트 | 역할 | 주요 산출물 |
| --- | --- | --- |
| **PM** | 제품 전략 수립, 기능 우선순위 결정 | PRD, 로드맵, KPI 정의 |
| **기획자** | 화면 기획, User Flow 설계 | 화면 기획안, 기능 명세서, 와이어프레임 |
| **디자이너** | UI 디자인, 컴포넌트 스펙 정의 | 디자인 스펙, 컴포넌트 스펙, Figma 파일 |
| **UX 라이터** | UI 문구 설계, 보이스 톤 관리 | UX 라이팅 가이드, 화면별 문구 시트 |

각 에이전트는 **SOLO 모드**(단독 호출)와 **CREW 모드**(오케스트레이터 연동) 두 가지로 동작합니다.

## 실행 모드

| 모드 | 설명 | 사용 상황 |
| --- | --- | --- |
| **FULL** | PM → 기획자 → 디자이너 → UX라이터 전체 실행 | 신규 서비스/대형 기능 출시 |
| **PLAN** | PM → 기획자 (기획까지만) | PRD + 화면 기획안이 필요할 때 |
| **DESIGN** | 기획자 → 디자이너 | 기획안이 이미 있고 디자인만 필요할 때 |
| **WRITE** | UX라이터 단독 | 문구 수정·UX 라이팅만 필요할 때 |
| **CUSTOM** | 에이전트를 직접 조합 | 특정 역할만 선택해서 호출할 때 |

## 사용 예시

아래는 실제 Claude Code에서 사용하는 프롬프트 형식과 각 에이전트의 산출물 예시입니다.

### FULL 모드 — 신규 기능 전체 프로세스

#### 프롬프트

```text
[서비스명] 앱에 '[기능명]' 기능을 새로 만들려고 해.
전체 프로세스 진행해줘.
```

#### 오케스트레이터 응답 흐름

```text
모드: FULL
실행 순서: PM → 기획자 → 디자이너 → UX라이터

Step 1. PM — PRD 작성 중...
  ✓ PRD 완료: workspace/[서비스명]/pm/PRD/PRD-...-v1.0.md
  ✓ 기능 목록 확정 (Must Have 3건 / Should Have 2건)

Step 2. 기획자 — 화면 기획안 작성 중...
  ✓ 화면 기획안 완료 (Figma)
  ✓ 기능 명세서 완료: workspace/[서비스명]/planner/feature-spec.md

Step 3. 디자이너 — UI 디자인 중...
  ✓ 디자인 스펙 완료: workspace/[서비스명]/designer/design-spec-v1.0.md
  ✓ 컴포넌트 스펙 완료

Step 4. UX라이터 — 문구 작성 중...
  ✓ UX 라이팅 가이드 완료
  ✓ 화면별 문구 시트 완료

작업 완료 보고 → session.md status: complete
```

### WRITE 모드 — UX 라이팅 단독 감수

#### 입력 예시

```text
[Figma URL] 이 화면 문구 전체 검토하고 수정해줘.
```

#### UX 라이터 산출물 예시

| 화면명 | 컴포넌트 | 원문 | 유형 | 수정안 | 수정 이유 |
| --- | --- | --- | --- | --- | --- |
| 로그인 | [에러] | 잘못된 비밀번호입니다 | B. 딱딱한 표현 | 비밀번호를 다시 확인해 주세요 | 사용자 탓하는 표현 제거 |
| 홈 | [빈상태] | 데이터가 없습니다 | C. 모호한 에러 | 아직 항목이 없어요. 추가해 보세요 | 다음 행동 안내 추가 |
| 저장 | [토스트] | 저장이 완료되었습니다 | B. 딱딱한 표현 | 저장했어요 | 능동태·간결체로 변환 |

## 실제 활용 사례

### 1. 차량 입고 등록 기능 전체 프로세스

위페어 파트너스 앱의 신규 기능을 FULL 모드로 실행. PM이 PRD를 작성하고 기능 우선순위를 확정한 뒤, 기획자가 AOS/iOS 화면 기획안을 Figma에 작성하고, 디자이너가 컴포넌트 스펙을 정의하고, UX 라이터가 현장 작업자 페르소나에 맞는 직관적인 문구로 마무리.

### 2. 기존 화면 UX 라이팅 감수

이미 Figma에 완성된 화면에서 금지 표현과 불일치 문구를 WRITE 모드로 검토. 화면별 문구 시트와 수정 이유를 Confluence 마크다운 형식으로 출력.

### 3. 보험 청구 플로우 화면 기획

PRD만 있는 상태에서 DESIGN 모드로 기획자→디자이너 두 에이전트만 호출. 관리자 페르소나 기준의 정보 밀도 높은 대시보드 레이아웃으로 화면 기획과 디자인 스펙을 한 세션에 완성.

### 4. 신규 서비스 PRD + 기획 빠른 초안

PLAN 모드로 PM→기획자 두 단계만 실행. 아이디어 단계에서 PRD 초안과 User Flow를 빠르게 뽑아 이해관계자에게 공유.

## 사용 기술

- **[Claude Code](https://claude.ai/code)** — 에이전트 런타임 및 오케스트레이터 실행 환경
- **CLAUDE.md** — 각 에이전트의 역할·인풋·아웃풋·작업 원칙 정의
- **GitHub** — 에이전트 설정 파일 버전 관리
- **Figma + Figma MCP** — 디자이너 에이전트가 화면을 직접 읽고 수정하는 연동
- **Confluence** — 각 에이전트 산출물의 최종 저장소 (마크다운 형식으로 작성)

---

## 앞으로의 계획

- **서비스 고도화에 실전 적용** — 신규 스프린트에 지속 투입하며 에이전트 정밀도 개선
- **세션 관리 자동화** — 오케스트레이터의 session.md 체크포인트를 기반으로 중단된 작업 자동 재개 강화
- **영업·마케팅 에이전트 확장** — 제품 조직을 넘어 세일즈 제안서, 마케팅 카피 등 조직 전체 업무로 확장
- **개발자 에이전트 추가** — UX 라이터 이후 구현까지 연결하는 개발자 에이전트 탐색

│   ├── full-process.md        # FULL 모드 워크플로우
│   ├── plan-process.md        # PLAN 모드 워크플로우
│   ├── quick-design.md        # DESIGN 모드 워크플로우
│   ├── ux-writing.md          # WRITE 모드 워크플로우
│   └── custom-flow.md         # CUSTOM 모드 워크플로우
└── workspace/
    └── [서비스명]/
        ├── pm/                # PRD, 로드맵
